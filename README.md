# 钉钉审批 PRD 生成器

🚀 一款自动从钉钉 OA 系统爬取审批表单并生成 PRD（产品需求文档）的工具。

## 项目功能

本工具用于从钉钉 OA 系统批量获取审批表单配置，并自动生成标准化的 PRD 文档，适用于以下场景：

- **系统迁移**：从钉钉审批迁移到自研系统时，快速梳理现有审批流程
- **需求文档化**：将散落在钉钉中的审批表单自动转换为标准 PRD 文档
- **流程审计**：批量导出审批流程配置，便于审计和存档
- **知识沉淀**：将审批表单结构化存储，便于团队理解和维护

### 核心特性

| 特性 | 说明 |
|------|------|
| 🔄 自动爬取 | 自动获取钉钉 OA 所有审批表单列表及详情 |
| 📝 PRD 生成 | 自动生成包含表单字段、审批流程的 PRD 文档 |
| 📊 CSV 导出 | 输出审批表单汇总清单（CSV 格式） |
| 💾 JSON 存储 | 保存原始表单数据（JSON 格式），便于二次开发 |
| ⚡ 并发处理 | 多线程并发获取，大幅提升效率 |
| 🔁 重新生成 | 支持从本地 JSON 文件重新生成 PRD（无需重新爬取） |

## 安装与配置

### 环境要求

- Go 1.21+
- 钉钉 OA 管理员权限（获取 Cookie）

### 编译安装

```bash
# 克隆项目
git clone <repository-url>
cd dingtalk-approval-prd-generator

# 编译
go build -o dingtalk_prd_generator .
```

### 配置文件

复制配置模板并填写：

```bash
cp config.yaml.example config.yaml
```

编辑 `config.yaml`：

```yaml
# 钉钉 Cookie（从浏览器开发者工具获取）
cookie: "你的钉钉Cookie"

# CSRF Token（从请求头 _csrf_token_ 获取）
csrf_token: "你的CSRF Token"

# 企业 ID（从请求头 x-client-corpid 获取）
corp_id: "你的企业ID"

# 输出目录
output_dir: "D:\\Download"

# 输出 CSV 文件名
output_filename: "approval_forms.csv"

# 并发工作线程数（建议 5-20）
workers: 10

# 失败重试次数
retry_count: 3
```

### 获取 Cookie 的方法

1. 登录钉钉 OA 网页版
2. 打开浏览器开发者工具（F12）
3. 切换到 Network 标签页
4. 刷新页面，找到任意请求
5. 从请求头中复制以下信息：
   - `Cookie` → 填入 `cookie` 字段
   - `_csrf_token_` → 填入 `csrf_token` 字段
   - `x-client-corpid` → 填入 `corp_id` 字段

## 使用方法

### 基本使用

```bash
# 运行程序（爬取并生成 PRD）
./dingtalk_prd_generator
```

程序将自动：
1. 获取所有审批表单列表
2. 并发获取每个表单的详细配置
3. 生成 CSV 汇总文件
4. 为每个表单生成 PRD 文档


## PRD 文档结构

每个 PRD 文档包含以下章节：

### 一、基本信息
- 表单名称、描述、类型
- 流程编码、状态、版本
- 最后修改人、修改时间
- 可见范围

### 二、表单字段设计
- 字段类型（自动翻译为中文）
- 字段名称、是否必填
- 字段说明（提示语、选项等）
- 字段联动规则

### 三、审批流程设计
- 流程图（ASCII 格式）
- 审批/抄送/审核节点
- 条件分支逻辑
- 会签/或签标注
